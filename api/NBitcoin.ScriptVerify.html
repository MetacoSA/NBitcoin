CheckSequenceVerify

CleanStack

Require that only a single stack element remains after evaluation. This changes the success criterion from "At least one stack element must remain, and when interpreted as a boolean, it must be true" to "Exactly one stack element must remain, and when interpreted as a boolean, it must be true". (softfork safe, BIP62 rule 6) Note: CLEANSTACK should never be used without P2SH.
Consensus

Strict consensus script verification flags that are checked by the Bitcoin Core consensus library before validating the transaction.
DerSig

Passing a non-strict-DER signature to a checksig operation causes script failure (softfork safe, BIP62 rule 1)
DiscourageUpgradableNops

Discourage use of NOPs reserved for upgrades (NOP1-10)

Provided so that nodes can avoid accepting or mining transactions containing executed NOP's whose meaning may change after a soft-fork, thus rendering the script invalid; with this flag set executing discouraged NOPs fails the script. This verification flag will never be a mandatory flag applied to scripts in a block. NOPs that are not executed, e.g. within an unexecuted IF ENDIF block, are not rejected.
DiscourageUpgradableWitnessProgram

Making v2-v16 witness program non-standard
ForkId

Some altcoins like BCash and BGold requires ForkId inside the sigHash
LowS

Passing a non-strict-DER signature or one with S > order/2 to a checksig operation causes script failure (softfork safe, BIP62 rule 5).
Mandatory

Mandatory script verification flags that all new blocks must comply with for them to be valid. (but old blocks may not comply with) Currently just P2SH, but in the future other flags may be added, such as a soft-fork to enforce strict DER encoding.

Failing one of these tests may trigger a DoS ban - see CheckInputs() for details.
MinimalData

Require minimal encodings for all push operations (OP_0... OP_16, OP_1NEGATE where possible, direct pushes up to 75 bytes, OP_PUSHDATA up to 255 bytes, OP_PUSHDATA2 for anything larger). Evaluating any other push causes the script to fail (BIP62 rule 3). In addition, whenever a stack element is interpreted as a number, it must be of minimal length (BIP62 rule 4). (softfork safe)
MinimalIf

Segwit script only: Require the argument of OP_IF/NOTIF to be exactly 0x01 or empty vector
None
NullDummy

verify dummy stack item consumed by CHECKMULTISIG is of zero-length (softfork safe, BIP62 rule 7).
NullFail

Signature(s) must be empty vector if an CHECK(MULTI)SIG operation failed
P2SH

Evaluate P2SH subscripts (softfork safe, BIP16).
SigPushOnly

Using a non-push operator in the scriptSig causes script failure (softfork safe, BIP62 rule 2).
Standard

Standard script verification flags that standard transactions will comply with. However scripts violating these flags may still be present in valid blocks and we must accept those blocks.
StrictEnc

Passing a non-strict-DER signature or one with undefined hashtype to a checksig operation causes script failure. Passing a pubkey that is not (0x04 + 64 bytes) or (0x02 or 0x03 + 32 bytes) to checksig causes that pubkey to be + skipped (not softfork safe: this flag can widen the validity of OP_CHECKSIG OP_NOT).
Witness

Support segregated witness
WitnessPubkeyType

Public keys in segregated witness scripts must be compressed
